# **唱片机冬令营课程**

## 课程总览

**核心理念**：在有限时间内完成“可演示的作品”，聚焦核心技能，简化流程，确保每个孩子都能带着成果回家。

------

## 调整策略

1. **功能优先级**：确保基础功能（读卡→播放音乐→亮灯）绝对可靠
2. **流程简化**：3D外壳改为激光切割/手工材料组装
3. **代码准备**：核心代码预先生成，学生以“修改参数”为主
4. **小组协作**：2人一组，明确分工（硬件员/软件员）
5. **每日目标**：每节课必须完成一个可验证的里程碑

------

## 详细课表（5天×2小时）

### **Day 1：认识电子伙伴 & 点亮第一盏灯**

**主题**：硬件初体验与基础电路
**目标**：完成所有硬件连接，实现LED控制

**时间安排**：

- **3:00-3:20** 快速破冰 + 项目展示（成品演示激发兴趣）
- **3:20-3:50** 认识元件 + 面包板连线（ESP32、LED灯带、电源）
  - 教师示范一组，学生模仿一组
  - 使用彩色接线图（不同颜色对应不同功能）
- **3:50-4:30** 第一段代码：让灯带亮起来
  - 安装Arduino IDE（提前准备离线安装包）
  - 上传预写好的LED测试程序
  - 修改颜色参数（个性化调色）
- **4:30-5:00** 巩固练习 + 挑战任务
  - “呼吸灯”效果尝试
  - 检查所有小组的LED是否正常工作

**课后准备**：

- 学生在家下载喜欢的3首音乐（提前通知）
- 为每组的SD卡预装音乐文件（助教完成）

------

### **Day 2：魔法卡片与音乐盒**

**主题**：RFID功能实现与音乐播放
**目标**：实现刷卡播放音乐的基础功能

**时间安排**：

- **3:00-3:25** RFID原理简介 + 硬件连接
  - 增加RC522模块、DFPlayer模块接线
  - 重点讲解“电源共地”概念
- **3:25-4:00** 刷卡识别测试
  - 上传卡片扫描程序
  - 每组领取3张空白卡片，读取UID并记录
- **4:00-4:40** 音乐播放功能集成
  - 上传主程序（预配置好3首音乐）
  - 测试：刷卡→播放对应音乐→灯带变色
- **4:40-5:00** 功能验证与问题调试
  - “音乐卡片配对”游戏：将卡片与音乐关联
  - 常见问题快速解决指南

**简化处理**：

- 跳过“写卡程序”编写，使用预写好的卡片（或教师提前写好）
- 音乐文件命名规则简化：0001.mp3,0002.mp3, 0003.mp3

------

### **Day 3：3D建模实战**

**主题**：学习fusion 360建模，设计外壳
**目标**：完成可打印的外壳3D模型

**时间安排**：

- **3:00-3:30** Fusion360入门
  - 界面熟悉
  - 基础操作：绘图工具、缩放、组合、对齐、布尔运算
  - 学生练习：简单图形绘制以及布尔运算
- **3:30-4:30** 外壳主体设计
  - **步骤1：创建基础盒子**
    - 根据Day1测量的尺寸，设计容纳所有元件的盒子
    - 预留：USB接口孔
  - **步骤2：添加内部结构**
    - ESP32固定柱（带螺丝孔）
    - 电池仓位置（可选）
    - 导线通道
  - **步骤3：优化设计**
    - 倒角处理（防刮手）
    - 添加Logo或个性化文字
  - **教师提供模块化设计模板**，学生在此基础上修改
- **4:30-5:00** 设计评审与导出
  - 教师逐一检查设计的可打印性
  - 导出STL文件，统一提交
  - **今日成果**：每个小组一个可打印的3D模型

**夜间行动**：

- 教师/助教将所有设计文件切片
- 启动3D打印机，夜间连续打印
- 准备备用外壳（预防打印失败）

------

### **Day 4：美化装饰与功能优化**

**主题**：个性化设计与系统完善
**目标**：完成外观装饰，优化用户体验

**时间安排**：

- **3:00-3:40** 创意装饰工作坊
  - 绘画区：丙烯颜料、贴纸、装饰胶带（对外观进行个性化设计创作）
  - 功能区：添加开关、音量旋钮、指示标签
  - 结构区：加强关键部位，添加防滑垫
- **3:40-4:20** 功能个性化调整
  - 修改代码中的灯光颜色
  - 增加“欢迎音效”
  - 为每张卡片命名（“我的学习歌单”、“运动时刻”等）
- **4:20-5:00** 系统整合测试
  - 压力测试：连续刷卡10次
  - 用户体验测试：邀请其他小组试用
  - 最后调试与固定

**核心调整**：

- 强调“可靠性”，确保展示时不掉链子
- 准备备用方案（如某组作品故障，可临时借用教师演示机）

------

### **Day 5：成果展示与创意延伸**

**主题**：展示交流与未来展望
**目标**：完成作品展示，激发持续兴趣

**时间安排**：

- **3:00-3:30** 展示准备
  - 设计展示牌（项目名称、小组成员、功能特点）
  - 排练1分钟演示脚本
  - 设置展示台
- **3:30-4:30** 成果展示会（邀请家长参加）
  - 每组2分钟演示（1分钟讲解 + 1分钟功能展示）
  - 观众互动：让家长/其他学生刷卡体验
  - 拍照留念
- **4:30-5:00** 颁奖与未来展望
  - 奖项设置：最佳功能奖、最佳设计奖、最佳团队奖（每组都有奖）
  - 扩展创意讨论：“如果你还有一天时间，想增加什么功能？”
  - 资源分享：推荐学习网站、本地创客空间、后续课程
  - 颁发结营证书与项目资料包（代码、接线图、扩展想法）

## 课程精简要点总结

| 原10天内容   | 现5天简化方案              | 目的                   |
| :----------- | :------------------------- | :--------------------- |
| 详细编程教学 | 提供基础代码，修改参数为主 | 节省时间，确保成功     |
| 3D建模+打印  | 激光切割/手工制作          | 避免打印等待，即时成果 |
| 深入调试训练 | 预设常见问题解决方案       | 快速解决问题           |
| 多轮迭代优化 | 一次成功路径               | 保障成品率             |
| 复杂功能扩展 | 基础功能+简单个性化        | 控制复杂度             |

## 教师准备清单（关键事项）

1. **课前准备**（必须完成）
   - 所有硬件预先测试，确保100%可用
   - 预装Arduino环境（便携版放在U盘备用）
   - 准备好3套完整演示设备（教师用+备用）
   - 音乐文件预装到SD卡（每张卡3首音乐）
   - RFID卡片预先写入数据（1、2、3）
2. **每日流程控制**
   - 严格时间控制，每项活动设倒计时
   - “先完成的小组”帮助“未完成的小组”
   - 每天结束前10分钟清理整顿
3. **安全与备用方案**
   - 准备3个完全组装好的“应急套件”
   - 热熔胶枪、焊台等危险工具教师专用
   - 准备简易故障排查卡（图文并茂）
4. **成果保障**
   - 核心功能（读卡播放）必须在前两天完成
   - 外壳设计提供模板，避免“设计瘫痪”
   - 装饰材料丰富，确保美学上的成功感

------

## 学生收获预期

通过5天课程，学生将：

1. 制作出可工作的RFID音乐播放器（实物）
2. 理解“输入→处理→输出”的电子系统逻辑
3. 体验从设计到制作的全流程
4. 获得团队协作与问题解决的初级经验
5. 带着扩展学习资源，激发对科技制作的兴趣

------

**课程设计原则**：有限时间内最大化成就感，以“完成作品”为首要目标，技术深度为辅助目标。确保每个孩子离营时都能自豪地说：“这是我做的！”
